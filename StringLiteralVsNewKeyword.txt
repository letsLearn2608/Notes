String Literal

Example:

String s1 = "Hello";
String s2 = "Hello";


Key Points:

Memory Location:

Stored in String Pool (a special area in JVM memory).

JVM checks the pool first; if the string already exists, it reuses the reference.

Reference Check:

System.out.println(s1 == s2); // true


== returns true because both s1 and s2 point to the same object in the String Pool.

Efficiency:

Memory-efficient because duplicates are avoided in the pool.

Recommended for strings that repeat frequently.

2ï¸âƒ£ String using new Keyword

Example:

String s3 = new String("Hello");
String s4 = new String("Hello");


Key Points:

Memory Location:

Creates a new object in Heap memory (outside the String Pool).

Even if "Hello" exists in the String Pool, a new object is always created in Heap.

Reference Check:

System.out.println(s3 == s4); // false


== returns false because s3 and s4 are different objects, even though their content is the same.

Content Check:

System.out.println(s3.equals(s4)); // true


equals() checks actual string content, so it returns true.

Use string literals for constants and repeated values â†’ saves memory.

Use new String() only if you explicitly need a new object reference (rare case).

System.out.println(s1 == s2);      // false (different references)
System.out.println(s1.equals(s2)); // true  (same content)
